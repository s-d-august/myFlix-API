This is a REST API for an application called myFlix. It interacts with a database that stores info about various movies.

Endpoints

Business Logic URL HTTP Method Request Body Data Format Response Body Data Format
Return a list of all movies. /movies GET None A JSON Object holding data about all the movies.
Return data about a single movie by title. /movies/:title GET None A JSON object holding data (description, genre, director, image URL, whether it's featured or not) about a single movie.

Example: 
{
    Title: 'The Thing',
    Description: 'A shapeshifting alien invades a remote Antarctic research station.',
    Genre: 'Horror',
    Director: 'John Carpenter',
    Image URL: 'https://myflix.com/movies/images/the-thing'
    Featured: true
        }
Return data about a genre by name. /movies/genres/:genre GET None A plain text description of a single genre.

Example:

'Horror is a film genre that seeks to elicit fear or disgust in its audience for entertainment purposes. Horror films often explore dark subject matter and may deal with transgressive topics or themes. Broad elements include monsters, apocalyptic events, and religious or folk beliefs. (From Wikipedia.)'
Return data about a director by name. /movies/directors/:director GET None A JSON object holding data (bio, birth year, death year) about a single director.

Example:
{
    Name: 'John Carpenter',
    Bio: 'Lorum ipsum dolor',
    Birth Year: 1948,
    Death Year: null
        }
Allow new users to register. /users POST A JSON object holding data about the new user.

Example:
{
    Username: 'johndoe',
    Name: 'John Doe',
    Email: 'john.doe@test.com',
    Password: 'password'
        }
 A JSON object holding data about the new user, including an ID.

Example: 
{
    Username: 'johndoe',
    Name: 'John Doe',
    Email: 'john.doe@test.com',
    Password: 'password'
    ID: 5
        }
Allow users to update their info. /users/:username PUT A JSON object holding the data to be changed.

Example: {Username: 'johndoe'} 		A plain text message indicating that the info was changed.

Example: 'Username changed from "johndor" to "johndoe".'
Allow users to add a movie to their list of favorites. /users/:username/movies/:movieID POST A plain text object of the movie's title.

Example: 'The Thing'		 A plain text message indicating that the movie was added.

'Example: "The Thing" has been added to your favorites.'
Allow users to remove a movie from their list of favorites. /users/:username/movies/:movieID DELETE A plain text object of the movie's title.

Example: 'Bee Movie'		 A plain text message indicating that the movie was removed.

'Example: "Bee Movie" has been removed from your favorites.'
Allow existing users to deregister. /users/:username DELETE None A plain text message indicating whether the deregistration was successful.

'Example: User "johndoe" has been deleted.'
